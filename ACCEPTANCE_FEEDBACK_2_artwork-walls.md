## 一句话结论
32 幅画里 **26 幅现在是落地画架（应该挂墙）**，只有 6 幅真的贴到了墙。
而且落地那些画的中心高度（centerY）从 0.72 飘到 4.14，本该统一在 ~1.5m 齐眼高度。
**核心问题：挂墙判定太严（`MAX_WALL_FRAME_GAP = 0.1`），一旦画框中心离墙超过 0.1 个单位就放弃挂墙、
退化成落地画架，而这些世界大多是开放花园/水景，画选的位置背后根本没有近墙。**

---

## 名词对齐（避免误解）
- 「挂墙」= `freestanding:false`：画贴在一面竖直墙上，无支腿。**这是我们想要的。**
- 「落地画架」= `freestanding:true`：画立在两条支腿上戳在地上（`WorldLayer.js:265-273` 加支腿）。
  当前 26 幅都是这样，看起来像画斜立在地上/半空，很违和。
- `centerY` = 画面中心的世界 Y 高度。正常人眼高约 1.5m，所以**画中心应 ≈ 该处地面高度 + 1.5**。
- `wall` = 从画中心沿它的正面法线**反方向**（即画背后）射线 4 个单位内探到的最近墙距离；
  `none` = 4 单位内没有墙。

## 该怎么改（具体、可执行，不需要看画面）

### 修法 A（最小改动，优先做）：放宽挂墙吸附阈值 + 强制齐眼高度
1. **放宽 `MAX_WALL_FRAME_GAP`**（`src/render/WorldLayer.js:13`）从 `0.1` 提到约 `0.6`。
   实测很多画背后 1.4~3.7 处就有墙，只是超过 0.1 被判定"没贴住"。放宽后它们会吸附到那面墙。
   吸附时把画沿法线推到"离墙 `WALL_FRAME_GAP(0.04)`"处（`wallPlacementAt` 已有 `offset` 逻辑，:368）。
2. **搜墙距离**：`wallPlacementAt` 里探针 `colliderHit(origin, direction, 0.08, maxDistance)`（:360）
   的 maxDistance 用的是整房间对角线，够远；问题只在最后 `centerDistance < MIN(2.2)+GAP` 和
   `gap<=0.1` 两道卡（:365, :384）。把 :384 的 `MAX_WALL_FRAME_GAP` 放宽即可让"近处有墙"的画挂上。
3. **强制画中心齐眼**：不管挂墙还是落地，画中心 Y 都应 = 该 (x,z) 处地面高度 + `1.5`。
   现在画框内部把画钉在 local `y=1.48`（:220/225/230），但 frame 的 `groundY`（:261）取的是
   `groundHeightAt(x,z)+0.02`（:332）——地面高度一乱，整幅就飘（实测 centerY 0.72~4.57）。
   **改**：`placement.groundY` 统一改成"以该画所依附墙脚的地面高度为基准"，并且 clamp 画中心 Y 到
   `[地面+1.2, 地面+1.7]` 区间，杜绝 4.5m 高的画。

### 修法 B（治本）：给每个世界配"画墙锚点"，别再靠运行时猜
运行时对开放花园做射线找墙，本来就不稳。更稳的是在世界配置里**显式声明每个世界的画墙**。
在 `src/config/legacyAssets.js` 每个世界的 `profile` 旁加一组 `artWalls`（可选），例如：
```js
// 每面墙：一个锚点(x,z) + 法线朝向(画面朝visitor的方向, 弧度) + 该墙脚地面高度
artWalls: [
  { x: 4.6, z: -16.4, yaw: -Math.PI/2, groundY: 0.34 },  // 右墙
  { x: -1.2, z: -5.6, yaw: 0,          groundY: 0.34 },  // 前墙
  ...
]
```
然后放置逻辑优先用 `artWalls`：把第 i 幅画贴到第 i 面墙、中心高 = `wall.groundY + 1.5`、
朝向 `wall.yaw`、贴墙留 0.04 缝，`freestanding:false`。没配 `artWalls` 的世界才走现在的射线兜底。
- 对 **World 05 van-gogh 画廊**（唯一真封闭）最该先配，它四周都有实墙、最能立刻变好。
- 对 **World 01/04/08** 这类开放/无限空间，如果确实没有连续实墙，可以退一步：
  要么保留落地画架但**把高度和朝向修正齐**（中心 1.5m、正对通道、支腿落在地面上、不倾斜），
  要么给这些世界加一道简单的"展墙"几何专门挂画。

### 每个世界画作"放哪边"的建议（基于实测走向）
- **World 01 温室**：画现在都堆在中轴 x≈0.48 一条线、背后无墙。建议改成**沿路径左右两侧交替**
  （一幅偏左 x≈-2、一幅偏右 x≈+2），各自面向中轴通道，若两侧有温室框架就贴上去；否则做落地展板但对齐 1.5m。
- **World 02 花厅**：右侧 x≈+4.7 有墙（那幅已挂墙），把另外三幅也移到左右两侧墙面（左墙约 x≈-3、右墙 x≈+4.7），别留在中间 x≈-1.2。
- **World 03 水园**：画偏低浮在水上。移到岸边/廊柱一侧（实测 x≈-6.5 有墙），中心抬到 1.5m。
- **World 04 海岸别墅**：开放花园，画沿中轴飘。建议贴到别墅外墙/围廊一侧（x≈-6 附近实测 1.45 有墙），或做统一落地展板。
- **World 05 梵高画廊**：**封闭画廊，最优先**。左右两墙（实测背后 2.9~3.5 都有墙）各挂 2 幅，中心统一 1.5m。
- **World 06 阳光宫花园**：已 2 幅挂墙但偏高。把 4 幅都对到 z≈28 那面墙一线，中心降到 1.5m。
- **World 07 墨西哥庭院**：一幅挂到 4.57m 太高。四幅统一贴院墙（实测 x≈±0.9~7 一侧 1.5~1.8 有墙），中心 1.5m。
- **World 08 无限镜屋**：画随倾斜地面越走越高。把整行画的中心 Y **锁定为固定值**（如 1.5），不要跟着地面爬。

## 验收标准（改完这样自检，不用看画面）
在浏览器 console 跑（进入任一世界后）：
```js
// 统计当前世界所有画：挂墙数 / 落地数 / 中心高度范围
const wl=window.__MUSE_APP__.engine.worldLayer, V=window.__MUSE_APP__.engine.camera.position.constructor;
let mounted=0,stand=0,ys=[];
for(const r of wl.artworks.values()){
  const f=r.frame; f.updateWorldMatrix(true,true);
  const c=new V(0,1.48,0).applyMatrix4(f.matrixWorld);
  const fs=(f.userData.supports||[{visible:true}])[0].visible;
  fs?stand++:mounted++; ys.push(+c.y.toFixed(2));
}
console.log({mounted,stand,minY:Math.min(...ys),maxY:Math.max(...ys)});
```
目标：`stand`（落地）尽量为 0（开放世界可保留但需对齐），`minY/maxY` 都落在 **1.2~1.7** 之间
（现在是 0.72~4.57，全飘）。逐世界跑一遍，8 个世界都达标即通过。
